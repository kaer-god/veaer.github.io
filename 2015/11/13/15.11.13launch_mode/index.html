Activity启动模式

Activity的LaunchMode

对于大部分刚接触 Android 的人，好像并不大会关注 Activity的启动模式。

android:launchMode

当前 Activity 启动另一个 Activity 时，新的Activity 会被推送到堆栈顶部，成为焦点所在。

前一个 Activity 仍保留在堆栈中，但是处于停止状态。

Activity 停止时，系统会保持其用户界面的当前状态。

用户按“返回”按钮时，当前 Activity 会从堆栈顶部弹出（Activity 被销毁），而前一个 Activity 恢复执行（恢复其 UI 的前一状态）。

堆栈中的 Activity 永远不会重新排列，仅推入和弹出堆栈：启动新的 Activity 将其推入堆栈；用户使用“返回”按钮退出时，Activity 销毁弹出堆栈。

因此，返回栈以“后进先出”对象结构运行。

Activity 的启动模式有四种：standard、singleTop、singleTask、singleInstance。

  1. standard:

    这是 Activity 默认的启动模式，你可以不用在清单文件中专门指明这个模式，系统在启动 Activity 的任务中创建 Activity 的新实例并向其传送 Intent，将其置于栈顶。Activity 可以多次实例化，而每个实例均可属于不同的任务，并且一个任务可以拥有多个实例。

  2. singleTop:

    如果你要启动的 Activity 并没有处于当前栈顶，就会启动一个新的 Activity。

  3. singleTask:

    如果你要启动的 Activity 存在于当前任务栈中，不论是否处于栈顶，都会直接唤起当前已经存在的 Activity 而不会去新建，并且还会将这个 Activity 到栈顶之间的所有 Activity 都出栈。

  4. singleInstance:

    这个模式有点『特殊』，之前的三种启动模式，无论是新建，还是从栈里寻找，都是基于一个栈的，如果启动模式设置为singleInstance，那么这个新建的 Activity 会出现在一个新的栈里。

    举个栗子，现在有三个 activity：A/B/C，其中 B 设置为 singleInstance，A/C为默认 standard，启动顺序为 A 启动 B，B 启动 C，那么当前页面在C 上面，现在按下back 按键，你会发现很神奇的 C 销毁之后，出现的并不是B，而是 A，当 A 再销毁之后，出现的才是 B。因为 A/C 是在同一个栈中的，C 销毁（出栈）之后，A 就露出了。

除了可以在AndroidManifest.xml文件中指定启动模式之外，在 java 代码中也可以。

Activity的flags

  1. Intent.FLAG_ACTIVITY_NEW_TASK

    作用和standard 相同

  2. Intent.FLAG_ACTIVITY_SINGLE_TOP

    作用和singleTop 相同

  3. Intent.FLAG_ACTIVITY_CLEAR_TOP

    作用和singleTask 相同

  4. Intent.FLAG_ACTIVITY_EXCLUDE_FROM_RECENTS

    有此标识的 Activity 不会出现在历时列表里
